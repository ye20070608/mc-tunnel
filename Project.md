项目设计方案：一体化MC服务器穿透控制软件

1. 项目背景与目标

1.1 背景

许多MC服主希望在自己的本地电脑上搭建随时在线的服务器，但受限于家庭宽带无公网IP、运营商端口封锁等问题。现有内网穿透工具（如frp、SakuraFrp）配置复杂，且需要自行管理多个端口和服务。本项目旨在开发一款一站式软件，将MC服务端、内网穿透、Web介绍页和管理后台整合，降低搭建门槛，并提供可视化的端口开放控制。



1.2 目标

提供 Web 管理后台作为图形界面，无需额外安装桌面客户端；同时保留命令行模式供高级用户使用。用户可一键启动/停止MC服务。



内置内网穿透客户端，支持选择开放端口（MC端口、介绍页端口、管理后台端口）。



提供基于Web的服务器介绍页（端口2）和管理后台（端口3），管理后台具备登录鉴权，未登录或登录失败时自动重定向至介绍页。



对非MC协议的请求（如HTTP访问到MC端口）进行智能识别或隔离，避免端口混淆。



2. 系统架构

2.1 总体架构

软件运行于用户本地电脑（Windows/Linux），由以下组件构成：



控制核心（Controller）：负责管理各子服务的生命周期，协调穿透配置。



MC服务端：独立的Minecraft服务进程（如Paper、Forge），监听本地端口（如25565）。



内网穿透客户端（Tunnel Client）：集成frp或类似工具，根据用户配置创建隧道，将本地端口映射到公网。支持TCP和UDP两种协议（UDP用于后续基岩版MC支持）。



内置Web服务器：提供两个站点：



介绍页站点：静态或动态页面，展示服务器信息、规则、玩家动态等。



管理后台站点：提供登录页面和后台管理功能（如重启服务、查看日志、修改白名单等）。



流量路由/过滤模块：前置TCP/UDP代理层，用于区分MC协议与HTTP流量，对非MC协议的请求进行拦截或重定向，保证端口隔离。同时预留UDP代理能力，为后续支持基岩版（Bedrock）提供扩展空间。

3. 功能模块设计

3.1 控制核心

服务管理：启动/停止/重启MC进程，监控进程健康状态。



配置管理：读取用户设定的端口映射关系、穿透服务器地址、鉴权信息等，生成frp配置文件并动态更新。



穿透客户端管理：启动/停止frp子进程，并监控其连接状态。



日志收集：汇总MC日志、frp日志、Web访问日志，提供统一查看接口。



3.2 内网穿透模块

集成成熟的开源穿透客户端（如frp），支持TCP和UDP协议（UDP用于后续基岩版MC支持）。



支持自定义穿透服务器（用户可选用自己的VPS或默认服务商）。



自动重连机制（由 frpc 自身处理），网络波动后尝试恢复隧道。



端口映射规则可动态添加/删除，无需重启软件。



UDP端口映射：预留配置项，用于后续基岩版MC服务端穿透（Bedrock默认使用UDP 19132端口），一期实现TCP，二期补全UDP支持。



3.3 Web服务模块

介绍页（端口2）：



技术选型：轻量级HTTP服务器（内嵌于控制核心，或使用Caddy）。



内容：支持Markdown或HTML静态页面，可配置服务器状态（在线人数、MOTD等）。介绍页通过少量JavaScript轮询控制核心提供的聚合API（本地回环）获取实时状态，无需直接访问MC端口。



管理后台（端口3）：



提供用户登录界面（支持多管理员账号，账号密码存储于本地配置文件）。



登录成功后进入管理面板，包含：



服务器启停控制。



在线玩家列表与踢人操作，含所在世界（维度和坐标）、在线时长追踪、一键设置/撤销 OP。



白名单管理（添加/删除玩家）。



查看实时日志。



修改穿透端口映射。



密码修改功能。



操作日志查看（记录所有敏感操作的操作人、时间、IP和内容）。



未登录访问管理页面时，自动302重定向到介绍页（端口2）。对于API请求（Accept: application/json 或 /api/ 前缀），未认证返回401 JSON响应（而非302），避免混淆。所有管理后台API端点必须经过鉴权中间件，携带有效Session或JWT Token。增加CSRF Token校验机制防止跨站请求伪造。



3.4 流量识别与重定向逻辑

端口1（MC端口）— 前置TCP代理层：

该端口只应接收MC协议流量。在控制核心中内建一个轻量级TCP代理，对每条新连接的前1-2字节进行协议嗅探：
- 首字节为 0xFE（旧版MC握手）或符合VarInt编码的长度前缀 → 判定为MC协议，透传至MC服务端。
- 首字节为 ASCII 字母（如 GET / POST / HEAD） → 判定为HTTP请求，直接关闭连接或返回HTTP 302重定向至介绍页（避免暴露服务）。
- 无法识别的字节序列 → 关闭连接并记录告警日志。

该代理层同时预留UDP数据报处理能力，为后续支持基岩版MC提供扩展。

端口3（管理后台端口）：
- 页面请求（Accept: text/html）：未认证 → 302重定向到介绍页（端口2）。
- API请求（Accept: application/json 或 /api/ 前缀）：未认证 → 返回401 Unauthorized（JSON格式），不含重定向。
- 所有API请求需携带CSRF Token（从页面获取，SameSite Cookie策略）。
- 重定向目标URL可通过配置文件修改。



4. 技术选型建议

主推方案：Go 语言（控制核心 + Web服务器统一技术栈，编译为单文件无运行时依赖，便于分发给普通用户）。
备选方案：Python + Flask（若团队主要熟悉Python，可用 PyInstaller 打包为单个exe，但体积较大）。

组件	技术方案	理由

控制核心	Go (gin/fiber) 或 Python + Flask	Go编译为单文件无依赖；Python需PyInstaller打包，但开发速度快。

MC服务端	PaperMC（推荐）或官方原版	性能优秀，支持插件。

穿透客户端	frp（开源）	成熟稳定，支持TCP/UDP，配置灵活。

Web服务器	Go net/http 内嵌 或 Caddy	统一技术栈（Go方案），减少外部依赖。

管理后台	Vue.js + 内嵌API 或 Alpine.js+HTMX	现代化界面，便于交互。一期可用轻量方案快速交付。

鉴权	本地YAML/JSON文件存储，BCrypt加密	支持多用户，无需外部数据库。

日志	Go: slog/zap；Python: Loguru	结构化日志，分级查看。



5. 接口与数据结构

5.1 配置文件（config.yaml）

yaml

# MC服务端配置
mc:
  version: "1.20.4"
  port: 25565
  max_players: 20
  java_path: "C:\\Program Files\\Java\\jdk-17\\bin\\java.exe"  # JDK路径
  jvm_args: "-Xmx4G -Xms2G"  # JVM启动参数
  auto_restart: true           # 崩溃后自动重启
  restart_max_retries: 3       # 最大重启重试次数

# Web服务配置
web:
  intro_port: 8080
  admin_port: 8443
  session_timeout: 3600        # Session过期时间（秒）
  csrf_enabled: true
  rate_limit: "10/minute"      # 登录频率限制

# 管理员账号（支持多用户）
admins:
  - username: "admin"
    password_hash: "$2b$12$..."  # BCrypt
  - username: "op"
    password_hash: "$2b$12$..."

# 穿透配置
tunnel:
  server_addr: "frp.example.com"
  server_port: 7000
  token: "your_token"
  protocol: "tcp"              # tcp / udp / both
  mapping:
    mc:
      local_port: 25565
      remote_port: 25565
      protocol: "tcp"          # Java版用TCP
    intro:
      local_port: 8080
      remote_port: 8080
      protocol: "tcp"
    admin:
      local_port: 8443
      remote_port: 8443
      protocol: "tcp"
    # 基岩版UDP预留（二期）
    # bedrock:
    #   local_port: 19132
    #   remote_port: 19132
    #   protocol: "udp"
  enabled_ports: ["mc", "intro", "admin"]  # 用户选择开启哪些

5.2 内部API

控制核心提供本地HTTP接口供管理后台调用，如：



POST /api/mc/start、/stop、/restart



GET /api/mc/status（在线人数、TPS等）



GET /api/mc/players（在线玩家列表）



GET /api/logs（返回最近日志）



POST /api/whitelist/add、/remove



POST /api/tunnel/update（修改映射配置）



POST /api/admin/change-password（修改管理员密码）



GET /api/admin/operation-log（操作日志查询）



这些API只监听本地回环（127.0.0.1），管理后台通过内网访问。所有API端点均需鉴权中间件校验。



6. 安全设计

管理后台鉴权：使用Session或JWT，默认强制HTTPS（若穿透支持）。API请求与页面请求分开处理——页面未认证返回302，API未认证返回401。



多管理员支持：配置文件支持多个管理员账号，各自独立密码（BCrypt加密存储）。



防止暴力破解：登录失败超过阈值（可配置，默认10次/分钟）后临时锁定IP并增加响应延迟。



CSRF防护：管理后台所有状态变更请求需携带CSRF Token，SameSite Cookie策略。



MC端口过滤：通过前置TCP代理层进行协议嗅探，拒绝非MC协议请求，避免端口被滥用。



穿透通信加密：frp支持TLS加密，应启用。



密码存储：使用BCrypt哈希，不明文存储。提供密码修改API，需旧密码验证。



操作日志：所有敏感操作（启停服务、修改白名单、修改配置等）记录到日志文件，包含操作人、时间、来源IP和操作详情。



最小权限原则：控制核心以普通用户权限运行，避免root。Web 服务绑定 127.0.0.1，仅 frp 对外暴露。


路径穿越防护：
  - 世界名输入：`validate_world_name()` 拒绝 `../`、`/`、`\`，仅允许 `[a-zA-Z0-9_-]+`
  - 日志导出：跳过 symlink，`resolve()` 验证路径在 `logs/` 内
  - 审计日志导出：限制到 `logs/exports/`，仅取文件名部分


输入校验：
  - 玩家名：`_validate_player_name()` 限 1-16 字符字母数字下划线，应用于白名单/踢人/OP 操作
  - 版本号：正则 `^[\d.]+$` 校验后入 glob，防 glob 注入
  - 路径原子写入：所有配置/server.properties 写入使用 `tempfile.mkstemp` + `os.replace` 防 TOCTOU 竞态


7. 部署与运维

7.1 安装流程

用户下载安装包（Go编译为单文件，或Python PyInstaller打包的exe）。



首次启动时引导配置：选择MC版本、穿透服务器信息、管理员账号密码。



EULA确认：若软件自动下载MC服务端核心，需强制用户确认Mojang EULA协议；或提供手动指定已有服务端JAR路径的选项。



软件可自动下载MC服务端核心（需用户确认EULA），也可使用用户已有的服务端JAR。



启动前自动检测端口占用和Java版本兼容性（Java 17+ 用于1.20+版本），若不符则提示用户。



用户选择要开放的端口（默认全部开启）。



一键启动，所有服务自动运行。



7.2 更新与维护

支持在线更新MC服务端核心和软件本体。



提供日志导出功能，便于故障排查。



提供一键备份世界和配置文件。



7.3 监控与告警

控制核心定期检测各子进程状态，若异常退出则自动重启（可配置重试次数）。



穿透连接断开时，记录日志并尝试重连。



8. 非功能性需求

性能：控制核心资源占用低，不影响MC性能。



跨平台：支持Windows 10+、Ubuntu 20.04+、macOS（可选）。



可用性：Web管理后台即图形界面，无需额外桌面客户端；同时保留命令行模式供高级用户和服务器环境使用。



可扩展性：支持后续添加更多端口映射（含UDP）、更换穿透工具、扩展更多管理功能。

10. 风险与应对

风险	应对措施

穿透服务商不稳定	支持自定义frp服务器，用户可自建；预设多个frp服务端地址做故障切换。

frp服务端被墙/限速	支持多节点自动切换，当主节点不可用时切换到备用节点。

MC服务端崩溃	监控并自动重启，保留崩溃日志。

管理后台密码遗忘	提供通过配置文件重置密码功能。

端口冲突	安装前检测端口占用，提示用户修改；支持自定义端口。

Windows Defender误报	提前将编译产物提交至主流杀软厂商白名单审核；提供开源源码供用户自行编译。

Java版本不兼容	启动前检测Java版本，不兼容时给出明确提示和下载链接。

法律/合规风险	自动下载MC服务端JAR时强制EULA确认；明确仅限非商业用途，用户需遵守当地法规。



11. 结论

本项目通过集成穿透、服务管理和Web界面，为MC服主提供了一站式解决方案。设计上注重易用性、安全性和灵活性，满足小规模社区服的需求。v1.1修订版根据技术评审意见，对流量识别方案、鉴权体系、技术选型、配置文件结构和风险矩阵进行了全面增强。后续可根据用户反馈迭代增强功能，如支持插件管理、地图上传、基岩版UDP穿透等。
