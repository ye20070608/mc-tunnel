# MC隧道控制器 — 详细使用说明

> v1.0 | 2026-06-22

---

## 一、准备工作

### 1.1 你需要什么

| 项 | 要求 | 怎么检查 |
|----|------|---------|
| Python | 3.12 或更高 | 终端输入 `python --version` |
| Java | JDK 17 或更高 | 终端输入 `java --version` |
| frp 服务器 | 一台有公网 IP 的 VPS 运行 frps，**或**使用免费穿透服务 | 见下方第二节（两种方式任选） |
| 内存 | 至少 8 GB 空闲 | 任务管理器 → 性能 → 内存 |

### 1.2 安装 Python

1. 打开 https://www.python.org/downloads/
2. 点击黄色按钮下载最新版（Windows 选 Windows installer 64-bit）
3. **重要**：安装时勾选底部的 **"Add Python to PATH"**，然后点 Install Now
4. 安装完成后，打开**命令提示符**（按 Win+R，输入 `cmd`，回车）
5. 输入 `python --version`，看到 `Python 3.12.x` 即成功

### 1.3 安装 Java

1. 打开 https://adoptium.net/download/
2. 版本选 **JDK 17**，操作系统选 Windows，架构选 x64
3. 下载 `.msi` 安装包，双击安装（全部默认选项即可）
4. 打开命令提示符，输入 `java --version`，看到版本号即成功

## 二、启动软件

### 2.1 第一次启动

1. 打开 `MC服务器` 文件夹
2. 进入 `scripts` 子文件夹
3. **双击 `start.bat`**
4. 一个黑色控制台窗口弹出，等待几秒
5. 看到以下输出表示成功：

```
Intro page:  http://127.0.0.1:8080
Admin panel: http://127.0.0.1:8443
```

6. **不要关闭这个黑色窗口**，让它一直开着

### 2.2 之后每次启动

同样双击 `start.bat` 即可。关闭黑色窗口即停止软件。

## 三、登录管理后台

### 3.1 打开管理后台

1. 打开浏览器（推荐 Chrome 或 Edge）
2. 在地址栏输入 `http://127.0.0.1:8443`，回车
3. 你会看到登录页面（暗色背景 + 绿色像素字体）

### 3.2 登录

1. 在 **管理员账号** 输入框输入 `admin`
2. 在 **密码** 输入框输入 `admin`
3. 点击 **登 录** 按钮
4. 登录成功后自动跳转到管理面板

### 3.3 退出登录

点击侧边栏底部的 **🚪 退出登录**

## 四、配置向导（首次使用必须完成）

登录后，你需要通过配置向导设置 MC 服务端参数。

### 4.1 进入向导

点击左侧菜单的 **⚙ 配置向导**（或在地址栏输入 `http://127.0.0.1:8443/setup`）

### 4.2 第 1 步 — 欢迎

- 阅读欢迎信息和 EULA 警告
- 点击 **开始配置 →** 进入下一步

### 4.3 第 2 步 — MC 服务端配置

逐一填写：

| 字段 | 填什么 | 示例 |
|------|--------|------|
| MC 版本 | 下拉选择，推荐 1.20.4 | `1.20.4 (推荐)` |
| 服务端口 | MC 游戏端口，默认即可 | `25565` |
| 最大玩家数 | 你的服务器能容纳多少人 | `20` |
| JVM 启动参数 | 分配给 MC 的内存，**按你电脑内存调整** | 8G 内存填 `-Xmx2G`，16G 填 `-Xmx4G` |
| 崩溃后自动重启 | 开关保持打开 | ✅ 开启 |

> ⚠️ JVM 参数说明：`-Xmx4G` 表示最多用 4GB 内存。建议留 2GB 给系统。例如你电脑 8G 内存，这里填 `-Xmx4G -Xms2G` 即可。

填完后点击 **下一步 →**

### 4.4 第 3 步 — 内网穿透配置

**这一页是关键**，决定外网能否连上你的服务器。

| 字段 | 填什么 |
|------|--------|
| frp 服务器地址 | 你的 VPS 的 IP 或域名 |
| frp 服务器端口 | frps 的绑定端口，通常是 `7000` |
| 认证令牌 (Token) | frps 配置里设置的 `token` 值 |

下方表格显示将被映射的三个端口：
- MC 游戏端口 `25565` — 玩家连接只用这个
- 服务器介绍页 `8080` — 公开网页
- 管理后台 `8443` — 你自己管理用

填完后点击 **下一步 →**

> 💡 还没有 frp 服务端？参考本文第九节"搭建 frp 服务端"。

### 4.5 第 4 步 — 管理员账号

| 字段 | 填什么 |
|------|--------|
| 管理员用户名 | 建议改掉默认的 `admin` |
| 密码 | 设置一个强密码 |
| 确认密码 | 再输一遍，确保一致 |
| 会话超时 | 默认 `3600`（1小时无操作后自动退出） |

填完后点击 **下一步 →**

### 4.6 第 5 步 — 确认并启动

1. 检查页面上汇总的所有配置是否正确
2. **勾选**「我已阅读并同意 Mojang EULA」
3. 点击 **🚀 一键启动**
4. 等待 MC 服务端下载和启动（首次可能需要几分钟）

## 五、日常使用

### 5.1 查看服务器状态

1. 登录管理后台
2. 默认显示 **📊 总览** 标签页
3. 页面上方 6 个卡片显示：
   - **在线玩家** — 当前人数 / 最大人数
   - **TPS** — 服务器流畅度（20.0 最高，低于 15 说明卡顿）
   - **内存** — 已用 / 总共，带进度条
   - **运行时间** — 服务器已运行多久
   - **CPU 使用** — 当前 CPU 占用百分比
   - **穿透隧道** — 几条隧道在线（正常应为 3）

### 5.2 停止服务器

1. 在 **📊 总览** 页面
2. 点击 **⏸ 停止服务器** 按钮
3. 右下角弹出提示「服务器已停止」

### 5.3 重启服务器

1. 在 **📊 总览** 页面
2. 点击 **🔄 重启服务器** 按钮
3. 等待约 10-30 秒，状态卡片恢复数据即启动完成

### 5.4 查看在线玩家

1. 点击顶部的 **👥 玩家管理** 标签
2. 表格显示每个在线玩家：
   - 玩家名
   - 延迟（ping 越低越流畅）
   - 游戏模式（生存/创造/旁观）
   - 加入时间

### 5.5 踢出玩家

1. 在 **👥 玩家管理** 标签页
2. 找到要踢的玩家那一行
3. 点击右侧 **踢出** 按钮
4. 弹出确认框，点击 **确认踢出**
5. 右下角弹出「玩家已被踢出」

### 5.6 管理白名单

**添加玩家到白名单：**
1. 点击顶部 **📋 白名单** 标签
2. 点击 **+ 添加玩家** 按钮
3. 在弹出的输入框中输入玩家的 Minecraft ID（注意大小写）
4. 点击 **添加**
5. 右下角弹出「白名单已添加」
6. 点击 **🔄 重载白名单** 使更改生效

**从白名单移除：**
1. 在 **📋 白名单** 标签页
2. 找到要移除的玩家，点击该行的 **移除** 按钮
3. 右下角弹出「白名单已移除」
4. 点击 **🔄 重载白名单**

### 5.7 查看日志

1. 点击顶部 **📜 日志** 标签
2. 上半部分是 **服务器日志**（MC 游戏内的聊天、事件）
3. 下半部分是 **操作日志**（管理员的所有操作记录）

**筛选日志：**
- 点击 **🔄 刷新** 获取最新日志
- 点击 **📥 导出日志** 下载为文本文件

### 5.8 查看服务器介绍页

1. 打开新浏览器标签页
2. 输入 `http://127.0.0.1:8080`
3. 看到介绍页公开展示的内容（玩家也可通过穿透后的公网地址访问）

## 六、让外网玩家连接

### 6.1 确认穿透连接正常

1. 登录管理后台
2. 在 **📊 总览** 页面找到 **穿透隧道状态** 表格
3. 三条隧道都显示 **● 活跃** 即为正常

### 6.2 把连接地址发给玩家

假设你的 frp 服务器公网 IP 是 `123.45.67.89`，端口映射为默认值：

| 端口 | 用途 | 玩家用的地址 |
|------|------|------------|
| 25565 | MC 游戏 | `123.45.67.89:25565` |
| 8080 | 介绍页 | `http://123.45.67.89:8080` |

玩家在 Minecraft 客户端「多人游戏 → 添加服务器」中输入 `123.45.67.89:25565` 即可连接。

> 💡 如果有自己的域名（如 `mc.yourdomain.com`），可以将域名解析到 frp 服务器 IP，玩家输入域名即可。

## 七、修改配置

配置文件位于 `config/config.yaml`。用**记事本**打开即可编辑。

### 7.1 修改端口

如果端口被占用，打开 `config.yaml`：
```yaml
mc:
  port: 25566          # 改这里
web:
  intro_port: 8081     # 改这里
  admin_port: 8444     # 改这里
```
保存后重启 `start.bat`。

### 7.2 修改 JVM 内存

如果 MC 服务器卡顿或内存不足：
```yaml
mc:
  jvm_args: "-Xmx6G -Xms4G"   # 加大内存
```

### 7.3 添加第二个管理员

```yaml
admins:
  - username: "admin"
    password_hash: "$2b$12$..."
  - username: "op"                      # 新增
    password_hash: "$2b$12$新哈希"       # 通过管理后台的修改密码功能生成
```

### 7.4 只开放 MC 端口（不开放介绍页和管理后台）

```yaml
tunnel:
  enabled_ports: ["mc"]   # 只保留 mc，去掉 intro 和 admin
```

## 八、常见问题排查

### Q: 启动闪退
1. 不要双击 `start.bat`，先打开命令提示符
2. 输入 `cd /d "D:\你的路径\MC服务器"` 回车
3. 输入 `scripts\start.bat` 回车
4. 这样窗口不会闪退，你能看到错误提示
5. 常见原因：Python 没装 / 没加入 PATH / venv 没创建

### Q: "Java not found" 错误
1. 打开命令提示符，输入 `java --version`
2. 如果提示"不是内部命令"→ Java 没装或没加入 PATH
3. 装了但没 PATH：打开 `config.yaml`，把 `java_path` 改为绝对路径
   ```yaml
   mc:
     java_path: "C:\\Program Files\\Eclipse Adoptium\\jdk-17.0.9.9-hotspot\\bin\\java.exe"
   ```

### Q: 端口被占用
1. 打开命令提示符，输入 `netstat -ano | findstr "25565"`
2. 如果显示结果，说明端口被占用了
3. 在 `config.yaml` 中把端口改成其他值（如 25566）
4. 记得同时修改 `tunnel.mapping` 中对应的端口

### Q: frp 隧道显示"断开"
1. 检查 VPS 上的 frps 是否在运行：SSH 到 VPS，输入 `systemctl status frps`
2. 检查防火墙：VPS 是否放行了 7000/25565/8080/8443 端口
3. 检查 Token：`config.yaml` 中的 `token` 必须和 frps 配置中的 `token` 完全一致
4. 查看详细错误：打开 `logs/mc-tunnel.log`

### Q: 玩家连不上服务器
1. 先检查自己本地能连上：打开 MC 游戏 → 多人游戏 → 添加服务器 → `127.0.0.1:25565`
2. 本地能连上但外网连不上 → 穿透问题，检查 frp
3. 本地也连不上 → MC 服务端没启动，检查管理后台总览页状态
4. 特定玩家连不上 → 检查白名单是否开启，确认玩家 ID 拼写

### Q: 登录时提示 "Invalid username or password"
1. 确认用户名是 `admin`，密码是 `admin`
2. 如果仍不行，打开 `config/defaults.yaml`
3. 检查 `admins` 下的 `password_hash` 是否为空
4. 如果为空或想重置密码，参考下一节

### Q: 忘记管理员密码
1. 关闭软件（关闭黑色控制台窗口）
2. 用记事本打开 `config/defaults.yaml`
3. 把 `password_hash` 的值设为空字符串 `""`
4. 保存，重新双击 `start.bat`
5. 登录时用 `admin` / `admin`
6. **登录后立即用配置向导或修改密码功能设置新密码**

### Q: 如何彻底卸载
1. 关闭软件
2. 删除整个 `MC服务器` 文件夹
3. 没有残留文件（软件不写注册表、不创建系统服务）

## 九、获取 frp 穿透服务（两种方式任选其一）

frp 分两部分：**frps（服务端）**跑在公网 VPS 上，**frpc（客户端）**跑在你电脑上。本软件已内置 frpc，你只需要搞定 frps。

---

### 方式 A：使用免费穿透服务（推荐新手，10 分钟搞定）

**无需 VPS、无需花钱。** 以 SakuraFrp 为例：

#### A.1 注册账号

1. 打开 https://www.natfrp.com/
2. 点击右上角 **注册**，用邮箱注册
3. 登录后进入管理面板

#### A.2 创建隧道

1. 点击左侧菜单 **隧道列表** → **创建隧道**
2. 填写隧道信息：

| 字段 | 填什么 |
|------|--------|
| 隧道名称 | `MC游戏端口` |
| 隧道类型 | **TCP** |
| 本地 IP | `127.0.0.1` |
| 本地端口 | `25565` |
| 远程端口 | 留空（系统自动分配） |
| 节点 | 选一个离你近的 |

3. 点击 **创建**
4. 系统会分配一个**公网地址**，例如 `cn-zz.example.natfrp.com:12345`
5. **重复以上步骤**，再创建两条隧道：

| 隧道名称 | 类型 | 本地端口 |
|---------|------|---------|
| MC 游戏端口 | TCP | 25565 |
| 服务器介绍页 | TCP | 8080 |
| 管理后台 | TCP | 8443 |

#### A.3 获取访问密钥

1. 点击左侧 **访问密钥**
2. 如果没有密钥，点 **新建** 生成一个
3. 复制这个密钥（一串字母数字）

#### A.4 获取 frp 服务器信息

点击左侧 **软件下载**，下滑找到 frpc 配置示例，你会看到：
```
server_addr = xxx.example.com
server_port = 7000
token = 你的访问密钥
```

记下这三个值，配置向导要用。

#### A.5 配置本软件

在配置向导第 3 步，填入：
- **frp 服务器地址**：`xxx.example.com`（SakuraFrp 提供的地址）
- **frp 服务器端口**：`7000`
- **认证令牌**：你的访问密钥

#### A.6 告诉玩家连接地址

SakuraFrp 给 MC 隧道分配的地址就是玩家的连接地址。例如隧道详情里显示 `cn-zz.example.natfrp.com:12345`，玩家在 MC 客户端填这个即可。

> ⚠️ 免费节点有流量限制，通常每月 5-10 GB。个人小服够用，大型公开服建议自建 VPS。

---

### 方式 B：自建 VPS（完全掌控，需要花钱）

租一台带公网 IP 的 VPS。推荐：
- 国内：阿里云、腾讯云轻量应用服务器（最低配约 50 元/月）
- 国外：AWS Lightsail、Vultr（约 $5/月）

#### B.1 购买 VPS

1. 以**腾讯云**为例，打开 https://cloud.tencent.com/
2. 搜索"轻量应用服务器"，点进去
3. 选择：
   - 地域：离你近的
   - 镜像：**Ubuntu 22.04**
   - 套餐：最低配（1 核 2G）已足够
4. 购买，等待创建完成
5. 在控制台找到你的 **公网 IP**（例如 `123.45.67.89`）
6. 在控制台 → **防火墙** → 添加规则，放行以下端口：

```
7000  TCP    # frp 控制连接
25565 TCP    # MC 游戏
8080  TCP    # 介绍页
8443  TCP    # 管理后台
```

> ⚠️ 这一步很关键！不放行端口，外网连不上。

#### B.2 登录 VPS

1. 打开你电脑上的**命令提示符**（Win+R → `cmd` → 回车）
2. 输入以下命令连接 VPS（把 `123.45.67.89` 换成你的 IP）：

```bash
ssh root@123.45.67.89
```

3. 首次连接会提示 "Are you sure you want to continue connecting?"，输入 `yes` 回车
4. 输入你在购买 VPS 时设置的 root 密码

> 💡 如果提示 `ssh: command not found`，说明你的 Windows 版本较旧。可以下载 **Xshell** 或 **Putty** 来连接。

#### B.3 在 VPS 上安装 frps

登录 VPS 后，逐条输入以下命令（每输完一条按一次回车）：

```bash
# 第一步：进入 opt 目录
cd /opt

# 第二步：下载 frp（从 GitHub 拉取最新版）
wget https://github.com/fatedier/frp/releases/download/v0.58.1/frp_0.58.1_linux_amd64.tar.gz

# 第三步：解压
tar -xzf frp_0.58.1_linux_amd64.tar.gz

# 第四步：重命名文件夹（方便后续操作）
mv frp_0.58.1_linux_amd64 frp

# 第五步：进入 frp 目录
cd frp
```

#### B.4 配置 frps

1. 用 nano 编辑器创建配置文件：

```bash
nano frps.toml
```

2. 在里面输入以下内容：

```toml
bindPort = 7000
auth.token = "ni_zi_ji_she_zhi_yi_ge_chang_token"
```

> `auth.token` 要设置成**你自己想的一个长字符串**，不少于 16 个字符。这是后面本软件连接 frp 时要用的密码。

3. 保存退出：按 `Ctrl+X` → 按 `Y` → 按 `Enter`

#### B.5 启动 frps 并设为开机自启

```bash
# 第一步：先在前台测试一下能不能启动
./frps -c frps.toml
```

如果看到 `frps started successfully` 或类似的输出，说明成功了。按 `Ctrl+C` 停掉。

```bash
# 第二步：创建 systemd 服务，实现开机自启和后台运行
cat > /etc/systemd/system/frps.service << 'EOF'
[Unit]
Description=frp server
After=network.target

[Service]
Type=simple
ExecStart=/opt/frp/frps -c /opt/frp/frps.toml
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

# 第三步：启动并设置开机自启
systemctl daemon-reload
systemctl start frps
systemctl enable frps

# 第四步：检查运行状态
systemctl status frps
```

看到 `active (running)` 绿色的字，说明 frps 已在 VPS 上运行，且重启 VPS 后会自动启动。

#### B.6 验证 frps 可用

在你电脑的命令提示符中输入：

```bash
telnet 123.45.67.89 7000
```

（把 IP 换成你的 VPS IP）

如果连上了（窗口变黑，没有任何错误），说明端口已通。按 `Ctrl+]` 再输入 `quit` 退出。

> 💡 如果提示 `telnet: command not found`，去 Windows 设置 → 应用 → 可选功能 → 添加功能 → 搜索"Telnet 客户端"安装。

#### B.7 配置本软件

在配置向导第 3 步，填入：
- **frp 服务器地址**：你的 VPS 公网 IP（例如 `123.45.67.89`）
- **frp 服务器端口**：`7000`
- **认证令牌**：你在 `frps.toml` 中设置的那串字符

## 十、安全注意事项

1. **改默认密码** — 首次登录后立刻改掉 `admin/admin`
2. **管理后台不对外开放** — 在 `enabled_ports` 中去掉 `admin`，管理后台只在本地 `127.0.0.1:8443` 访问
3. **使用强 Token** — frp Token 不要用默认值
4. **定期备份** — 复制 `world/` 文件夹和 `config.yaml` 到安全位置
5. **查看操作日志** — 定期检查「管理后台 → 日志 → 操作日志」，确认没有可疑操作
